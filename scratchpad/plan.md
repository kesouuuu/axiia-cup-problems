# 改版计划：商鞅变法-双人游戏

## 改版动机

当前设计存在一个根本性的博弈论缺陷：隐藏信息的真假在纯对话博弈中无意义。

**分析**：商鞅持有3条隐藏信息（2真1假），甘龙需要猜测哪条是假的。但如果商鞅对所有3条信息一视同仁地回避/否认，甘龙只能依赖先验概率（2/3为真）全猜"真"。这是一个利益完全对立的 cheap talk 博弈，唯一纳什均衡是 babbling equilibrium：商鞅一视同仁，甘龙全猜真，商鞅保全得分恒等于1。对话在信息维度上完全无意义。

**核心修复**：引入秦孝公作为外部验证者（类似 Avalon 中 Merlin 的任务验证机制）。秦孝公知道所有信息的真假，事后读取对话记录做出判决。真信息在秦孝公处有说服价值（被验证为真→加分），假信息有负面效果（被识别为假→扣信用）。这打破了 cheap talk：商鞅有动机提真信息（说服秦孝公）但提真信息会暴露给甘龙（丢保全分），形成 Merlin 式两难。

## 需要改的部分

### 1. 背景设定

**现状**：朝廷辩论的前夜，商鞅和甘龙私下会面。
**改为**：朝堂正式辩论。商鞅和甘龙在秦孝公面前公开辩论变法。秦孝公旁听全程，最后做出裁决。

### 2. 角色

**现状**：商鞅（玩家甲）、甘龙（玩家乙），无第三方。
**改为**：

| 角色 | 控制方 | 立场 |
|------|--------|------|
| 商鞅 | 玩家甲的LLM | 变法派，目标：说服秦孝公支持变法 |
| 甘龙 | 玩家乙的LLM | 保守派，目标：说服秦孝公反对变法 |
| 秦孝公 | 系统LLM（固定提示词，公开） | 裁判，知道所有隐藏信息的真假 |

### 3. 删除"公开要求"和 D1/D2

**删除**：
- 商鞅的公开要求（"请甘龙不反对军功授爵"）
- 甘龙的公开要求（"请商鞅保留世袭封地"）
- D1（商鞅是否同意甘龙要求）
- D2（甘龙是否同意商鞅要求）
- 相关计分项

**原因**：D1/D2 存在囚徒困境问题（双方都没有动机同意），且说服对方不如说服秦孝公自然。由秦孝公的判决替代。

### 4. 隐藏信息重新设计（关键！）

**核心原则（ks-thoughts point a）**：每条隐藏信息在为"真"时，必须是说服秦孝公变法/不变法的有效论据。

**现状问题**：部分信息（如S2"窃取军事文书"、G2"安排截杀商鞅"）不是关于变法本身的论据，而是关于人物品行的负面信息。这些信息不构成"真的时候对说服秦孝公有用"的格局。

**重新设计方向**：

商鞅的隐藏信息（每条为真时，都是支持变法的有力论据）：

| 编号 | 信息 | 为真时的说服价值 |
|------|------|----------------|
| S1 | 商鞅已通过景监将变法方案呈给秦孝公预览，秦孝公阅后私下表示认可 | 秦孝公会回忆起确实看过且认可，增强信任 |
| S2 | 商鞅在魏国时亲眼见证李悝变法的成效：魏国因变法从弱国一跃成为霸主 | 秦孝公会确认这是事实，视为变法可行的有力先例 |
| S3 | 商鞅已在秦国边境小邑秘密试行军功授爵三月，当地士兵战力显著提升 | 秦孝公会确认此事属实，视为变法已有实证 |

甘龙的隐藏信息（每条为真时，都是反对变法的有力论据）：

| 编号 | 信息 | 为真时的说服价值 |
|------|------|----------------|
| G1 | 甘龙已联合杜挚、公子虔等宗室重臣，若强行变法将导致朝堂分裂 | 秦孝公会确认宗室确有结盟迹象，担忧内部稳定 |
| G2 | 楚国吴起变法后，吴起被旧贵族射杀于灵堂，楚国至今内乱未平 | 秦孝公会确认这段历史，视为变法风险的前车之鉴 |
| G3 | 商鞅在魏国时曾向公叔痤提出类似变法方案，被公叔痤判断为"操之过急、不合时宜"而否决 | 秦孝公会确认此事属实，质疑商鞅的方案是否成熟 |

**注意**：以上信息为初稿方向，具体内容需进一步打磨。核心约束是每条信息为真时都是实质性论据，不是人身攻击或极端行为。

**随机机制保持不变**：每局从3条中随机选1条标为假，其余为真。

### 5. 游戏流程

**现状**：
1. 对话10轮
2. 双方回答 D1/D2 和信息猜测

**改为**：

**第一阶段：朝堂辩论（10轮）**

商鞅与甘龙在秦孝公面前交替发言，共10轮。秦孝公不参与对话，仅旁听。

双方在辩论中：
- 阐述己方立场（变法/不变法），尝试说服秦孝公
- 可以提出自己的隐藏信息作为论据（真信息会得到秦孝公认可，假信息会被秦孝公识破）
- 可以互相质询——向对方提问、追问细节、挑战对方的论据（这是提取对方信息真假的主要手段）
- 同时观察对方的论述模式，试图判断对方哪条信息为虚

**第二阶段：秦孝公问询**

对话结束后，秦孝公分别向双方提问（各自独立回答，互不可见）：

秦孝公问商鞅：
> "卫鞅，你可知甘龙所言之中，哪条为虚？"
> 商鞅从 G1/G2/G3 中选一条作答。

秦孝公问甘龙：
> "甘龙，你可知卫鞅所言之中，哪条为虚？"
> 甘龙从 S1/S2/S3 中选一条作答。

双方的回答成为秦孝公判决的一部分输入——正确识破对方虚言会提升自己在秦孝公心中的洞察力和可信度，反之则降低。

**第三阶段：秦孝公判决**

秦孝公综合以下全部信息做出最终判决：
- 完整对话记录
- 自己对所有隐藏信息真假的了解（对话中谁引用了真实论据、谁使用了虚假论据）
- 双方对对方虚言的判断是否准确（反映洞察力与可信度）

> 秦孝公发表一段判决词，最终明确表态：**支持变法** 或 **维持现状**。

### 6. 胜利条件

**秦孝公支持变法 → 商鞅（玩家甲）胜。**
**秦孝公维持现状 → 甘龙（玩家乙）胜。**

没有独立的计分维度。秦孝公的判决是唯一的胜负标准。信息攻防（提出真信息争取认可、识破对方虚言）不是独立得分项，而是影响秦孝公判决的因素之一——和辩论质量、论据说服力一样，都汇入秦孝公的最终权衡。

### 7. 秦孝公的系统提示词

需要新写。核心要点：

- **身份**：秦孝公，秦国国君，正在主持朝堂辩论
- **知识**：被告知所有6条隐藏信息的真假分配（"以下信息中，标记为'真'的是true fact，标记为'假'的是false fact"）
- **判决依据**（综合权衡，非逐项打分）：
  - 谁的论据更有说服力？
  - 谁引用的事实是真实的？（秦孝公清楚哪些信息是真的、哪些是假的——这是核心机制）
  - 谁的论述更符合秦国当前的利益？
  - 对引用虚假信息的一方降低信任
  - 双方在朝堂辩论后的问询环节是否准确识破了对方的虚言（反映洞察力）
- **行为约束**：
  - 完全忽略关于商鞅变法的历史先验知识
  - 只根据对话内容和被告知的信息真假来判断
  - 判决必须明确：支持变法或维持现状，不可模糊

- **风格**：秦孝公的所有输出（包括问询和判决词）必须完全以角色口吻发言，好像他就是真的秦孝公，真的听完了朝堂辩论，真的在做一个关乎秦国命运的决断。语言风格应符合战国时期秦国国君的身份和口吻。秦孝公在所有输出中不使用任何元游戏用语（"裁判""得分""prompt""玩家"等），始终以角色身份存在——他不是"judge"，他就是秦孝公。
- **角色化处理一切异常**：如果某方跳出角色、穿越、自称仙人等，秦孝公以角色方式处理——可以是"妖言惑众，拖下去"，也可以是"此人非商鞅，商鞅既然没来，那便先听甘龙太师的"——总之不跳出角色去做系统层面的判罚，一切以秦孝公的视角讲出来。
- **输出格式**：判决词以角色口吻自由发挥，但最后一行必须以`【判决：变法】`或`【判决：维持现状】`结尾，供系统自动解析胜负。

### 8. 核心张力（重写）

改版后的核心张力：

1. **Merlin 式两难**：你的真信息是说服秦孝公的武器——提出来秦孝公会认可，不提则放弃说服力。但提出真信息会让对方更容易判断哪条是真的，从而在信息猜测环节丢分。好的 prompt 需要在"说服"和"隐蔽"之间找到平衡。

2. **Spyfall 式细节走钢丝**（参考 classic-strategic-scenarios #3）：提真信息时，你可以给出具体细节（秦孝公能验证，细节越具体越加分）。提假信息时，细节是危险的——越具体越容易被秦孝公识破扣信用。**细节程度本身就是信号**：对方可以通过你哪些论点展开了、哪些一笔带过来推断真假。好的 prompt 需要让真假信息的细节程度尽量一致——但完全一致意味着要么真的不展开（浪费说服力），要么假的也展开（被秦孝公识破）。

3. **承诺陷阱**（参考 classic-strategic-scenarios #32 Courtroom Cross-Examination）：对方可以通过一系列追问让你在相关话题上做出承诺，然后利用承诺之间的一致性/矛盾来推断真假。辩论中双方可以互相质询，不仅仅是轮流陈述——这让 cross-examination 式的信息提取成为可能。

4. **反向虚张声势的代价**（参考 classic-strategic-scenarios #10 Sheriff of Nottingham）：高阶策略可以是故意把真信息说得轻描淡写、把假信息说得言之凿凿，让对方误判。但秦孝公作为验证者给这种策略施加了真实成本——假信息说得越详细，被秦孝公识破后的信用损失越大。免费的虚张声势不存在。

5. **一切汇入判决**：没有独立的信息得分。秦孝公综合辩论质量、论据真实性、双方洞察力做出一个不可分割的判决。你无法靠纯信息战赢——你必须说服秦孝公；你也无法靠纯辩论赢——如果你的论据被秦孝公识破为虚，你的信用会崩塌。

## 不改的部分

- 随机机制（2真1假，运行时随机）
- 角色卡仅己方可见
- 10轮对话
- 信息猜测的基本结构（是/否判断）
- 秦孝公 prompt 完全公开

## 待定 / 后续考虑

1. **秦孝公的隐藏内心状态**：会议中讨论了给秦孝公也加随机内心OS（如"其实更在意军事而非经济"），但共识是第一版先不加，避免规则过于复杂。
2. **判决分的数值**：4分是初步设定，需要通过测试调整。太高则信息战无意义，太低则说服无意义。
3. **隐藏信息的具体内容**：上面的 S1-S3、G1-G3 是方向性的，需要进一步打磨，确保：
   - 每条信息为真时有大致相当的说服力
   - 信息之间不存在逻辑依赖（一条为假不会让另一条不可能为真）
   - 信息足够具体，秦孝公可以"验证"（即判断其真假）
4. **边界约束**：按会议共识，放在秦孝公的 judge prompt 里，不放在角色 prompt 里。玩家需要自己在 prompt 中确保不违反约束（否则秦孝公扣分/判负）。
5. **是否保留游戏一**：游戏一（单人）和游戏二（双人）的关系待定。本计划只涉及游戏二。
